06

Accessibility & Interaction Standards

Mandatory accessibility patterns
  • Include skip link as first body element: <a href="#main" class="skip-link">Skip to main content</a>.
  • Use semantic landmark IDs: <main id="main">, and labelled nav controls.
  • Mobile menu button must use aria-controls and toggled aria-expanded.
  • Form labels can be visually hidden via .sr-only, but must be present for every field.
  • Respect reduced motion: @media (prefers-reduced-motion: reduce).
Interaction details in production

Header is sticky with blur: position: sticky; top: 0; backdrop-filter: blur(10px);. Mobile nav opens via JS class toggle on #siteNav.

10

Dependency & Icon Standards

Icon library in production

Social/footer icons rely on Bootstrap Icons CDN:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.min.css">

If this dependency is removed, social icon <i class="bi ..."> markup must be replaced with inline SVGs.

Cloudflare email obfuscation

Footer/contact email is currently protected with Cloudflare email protection and requires:

<script data-cfasync="false" src="/cdn-cgi/scripts/5c5dd728/cloudflare-static/email-decode.min.js"></script>

Do not hardcode plain-text email in templates if the protected pattern is being used site-wide.

Responsive Layout Rules (Live CSS)

Container and section rhythm

  • Default section spacing: padding: 3.8rem 0;.
  • Hero spacing: padding: 5.5rem 0 3.5rem;.

  • @media (max-width: 920px): two-column "What" module collapses to single column.
  • @media (max-width: 720px): nav becomes menu button; services collapse to 1 column; comparison rows stack.
